Tällä sivulla ohjeita soRvi-paketin kehittäjille ja kontribuuttoreille. Sivu on työn alla ja ohjeita täydennetään pian.

  1. Asenna R ja soRvin vaatimat riippuvuudet
  2. Forkkaa klooni soRvin GitHub-kehitysversiosta
  3. soRvi-paketin kääntämisohjeet:

    Asenna devtools R:stä käsin: {% highlight r %} install.packages(devtools) {% endhighlight %}

    Kun soRvi-dev-kansio sijaitsee työhakemistossa, voit generoida dokumentaatiosivut R:ssä komentosarjalla {% highlight r %} library(devtools) document("soRvi-dev") {% endhighlight %}

    Käännä paketti komentosarjalla (korvaa myRpath R-asennushakemistolla): {% highlight r %} /myRpath/R CMD check soRvi-dev /myRpath/R CMD build soRvi-dev /myRpath/R CMD INSTALL sorvi_x.y.z.tar.gz {% endhighlight %}

    Korjaa muutoksiisi liittyvät virheilmoitukset ja varoitukset

  4. Kontribuoi
    Ylläpidon kannalta kätevin tapa kontribuutioiden vastaanottamiseen on GitHubin pull-request toiminto:

    Tee git-merge soRvin master branchiin

    Lähetä ylläpidolle pull-request

    Otamme vastaan kontribuutioita myös muilla tavoin. Jos sinulla on R-kielistä lähdekoodia, jonka olisit halukas kontribuoimaan soRvi-kirjastoon, ota yhteyttä ja katsotaan yhdessä miten voitaisiin edetä.

Lähdekoodi

Kehitysversion lähdekoodi löytyy Louhoksen github-sivulta.

Koodi on julkaistu avoimella FreeBSD-lisenssillä, joka sallii lähdekoodin vapaan käytön, muokkaamisen ja levittämisen, kunhan samat oikeudet jatkokäytölle taataan uusissakin tuotoksissa.
