Tällä sivulla ohjeita rOpenGov-pakettien kehittäjille. Kehityksen päävaiheet ovat kehitysversion kloonaus, paketin kääntäminen ja tarkistus, koodin kehittäminen, virheiden ja varoitusten korjaaminen, ja uusien kontribuutioiden liittäminen päähaaraan. Tiivis ohjeistus alla:

Kehitysversion kloonaus

  1. Asenna [git]() ja [R](http://www.r-project.org)
  2. Forkkaa klooni paketin GitHub-kehitysversiosta
  3. Asenna R:n devtools ja roxygen2-paketit.

Paketin kääntäminen ja asennus

    Lisättyäsi haluamasi funktiot ja niiden roxygen-dokumentoinnin uuteen tiedostoon paketin R-kansiossa, voit testata nämä seuraavasti:
  1. Dokumentaatiosivujen (sorvi/man/*.Rd) generointi R:ssä (olettaen, että sorvi sijaitsee työhakemistossa): {% highlight r %} library(devtools) document() {% endhighlight %}
  2. Paketin validointi (Windows-systeemeissä saatetaan tarvita LaTex-pakettien erillistä asentamista ennen checkiä): {% highlight r %} check() {% endhighlight %}
  3. Paketin kääntö: {% highlight r %} build() {% endhighlight %}
  4. Paketin asennus: {% highlight r %} install() {% endhighlight %}

Kontribuointi päähaaraan

Tee ensin kaikki kehitystyö omassa kehityshaarassasi:
  1. Korjaa muutoksiisi liittyvät virheilmoitukset ja varoitukset (build ja check antavat usein varoituksia)
  2. Varmista (git status-komennolla), että olet lisännyt (git add) uuden koodin paketin R-hakemistoon
  3. Päivitä ./NEWS.md (tiivistelmä muutoksista)
  4. Kätevin tapa kontribuutioiden vastaanottamiseen on GitHubin pull request-toiminto:
    • Tee git-merge omaan kehityshaaraasi sorvin master branchista
    • Lähetä ylläpidolle pull request
    Jos olisit halukas kontribuoimaan muilla tavoin, ota yhteyttä ja mietitään lisää.
  5. Lisää yksinkertaiset käyttöesimerkit paketin vignettes-kansion tutoriaaliin. Tämä on .Rmd-tiedosto, jonka voi kääntää markdowniksi R:ssä komennolla library(knitr); knit("sorvi_tutorial.Rmd").
  6. Laadi halutessasi blogikirjoitus, joka esittelee uuden toiminnon ja sen käyttöesimerkkejä. Voit kirjoittaa omaan blogiisi, tai ylläpitämäämme suomenkieliseen Louhos-blogiin tai englanninkieliseen rOpenGov-blogiin. Kirjoitusoikeuksia annetaan pyynnöstä.

Yhteisö ja tuki

Hanke on yhteisövetoinen avoimen lähdekoodin projekti. Pääkehittäjien yhteystiedot löydät täältä. Uudet tekijät ja palaute ovat tervetulleita.

Lähdekoodi

Kehitysversion lähdekoodi on julkaistu avoimella FreeBSD-lisenssillä, joka sallii lähdekoodin vapaan käytön, muokkaamisen ja levittämisen, kunhan samat oikeudet jatkokäytölle taataan uusissakin tuotoksissa.
