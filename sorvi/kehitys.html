Tällä sivulla ohjeita sorvi-paketin kehittäjille ja kontribuuttoreille. Sivu on työn alla ja ohjeita täydennetään pian. Sorvin kehitys tapahtuu GitHubissa. Kehityksen päävaiheet ovat (i) kehitysversion kloonaus; (ii) paketin kääntäminen; (iii) koodin kehittäminen; (iv) virheiden ja varoitusten korjaaminen; (v) uusien kontribuutioiden liittäminen päähaaraan. Tiivis ohjeistus alla:

Kehitysversion kloonaus

  1. Asenna R ja sorvin vaatimat riippuvuudet
  2. Forkkaa klooni sorvin GitHub-kehitysversiosta
  3. Asenna devtools R:stä käsin: {% highlight r %} install.packages(devtools) {% endhighlight %}

Paketin kääntäminen ja asennus

  1. Dokumentaatiosivujen (sorvi/man/*.Rd) generointi R:ssä (olettaen, että sorvi sijaitsee työhakemistossa): {% highlight r %} library(devtools) document("sorvi") {% endhighlight %}
  2. Paketin validointi: {% highlight r %} check("sorvi") {% endhighlight %}
  3. Paketin kääntö: {% highlight r %} build("sorvi") {% endhighlight %}
  4. Paketin asennus: {% highlight r %} install("sorvi") {% endhighlight %}

Kontribuointi päähaaraan

  1. Korjaa muutoksiisi liittyvät virheilmoitukset ja varoitukset
  2. Varmista, että olet lisännyt uuden koodin (pkg/R/*.R) kehityshaarasi alle
  3. Varmista, että olet lisännyt uudet dokumentaatiosivut (pkg/man/*.Rd) kehityshaarasi alle
  4. Päivitä pkg/DESCRIPTION (versionumero ja päivämäärä)
  5. Päivitä inst/NEWS (tiivistelmä muutoksista)
  6. Ylläpidon kannalta kätevin tapa kontribuutioiden vastaanottamiseen on GitHubin pull-request-toiminto, jossa teet kehitystyön omassa kehityshaarassasi, ja työn valmistuttua varmistat että se on yhteensopiva päähaaran (master branch) kanssa. Tämän jälkeen voit lähettää ylläpidolle pyynnön vetää muutokset päähaaraan. Tiiviit ohjeet:
    • Tee git-merge omaan kehityshaaraasi sorvin master branchista
    • Lähetä ylläpidolle pull-request esim. sorvin GitHub-sivuston kautta
    • Lisää ohjeita Gitin käyttöön sorvin kehityswikissä
    Otamme vastaan kontribuutioita myös muilla tavoin. Jos sinulla on R-kielistä lähdekoodia, jonka olisit halukas kontribuoimaan sorvi-kirjastoon, ota yhteyttä ja katsotaan yhdessä miten voitaisiin edetä.

Lähdekoodi

Kehitysversion lähdekoodi löytyy Louhoksen github-sivulta. Koodi on julkaistu avoimella FreeBSD-lisenssillä, joka sallii lähdekoodin vapaan käytön, muokkaamisen ja levittämisen, kunhan samat oikeudet jatkokäytölle taataan uusissakin tuotoksissa.
