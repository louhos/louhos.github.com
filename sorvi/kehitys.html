Tällä sivulla ohjeita soRvi-paketin kehittäjille ja kontribuuttoreille. Sivu on työn alla ja ohjeita täydennetään pian.

  1. Asenna R ja soRvin vaatimat riippuvuudet
  2. Forkkaa klooni soRvin GitHub-kehitysversiosta
  3. Asenna devtools R:stä käsin: {% highlight r %} install.packages(devtools) {% endhighlight %}
  4. Dokumentaatiosivujen generointi R:ssä (olettaen, että soRvi-dev sijaitsee työhakemistossa): {% highlight r %} library(devtools) document("soRvi-dev") {% endhighlight %}
  5. Paketin validointi: {% highlight r %} check("soRvi-dev") {% endhighlight %}
  6. Paketin kääntö: {% highlight r %} build("soRvi-dev") {% endhighlight %}
  7. Paketin asennus: {% highlight r %} install("soRvi-dev") {% endhighlight %}
  8. Korjaa muutoksiisi liittyvät virheilmoitukset ja varoitukset
  9. Kontribuoi. Ylläpidon kannalta kätevin tapa kontribuutioiden vastaanottamiseen on GitHubin pull-request toiminto:

    Tee git-merge soRvin master branchiin

    Lähetä ylläpidolle pull-request

    Mutta otamme ilman muuta vastaan kontribuutioita myös muilla tavoin. Jos sinulla on R-kielistä lähdekoodia, jonka olisit halukas kontribuoimaan soRvi-kirjastoon, ota yhteyttä ja katsotaan yhdessä miten voitaisiin edetä.

Lähdekoodi

Kehitysversion lähdekoodi löytyy Louhoksen github-sivulta.

Koodi on julkaistu avoimella FreeBSD-lisenssillä, joka sallii lähdekoodin vapaan käytön, muokkaamisen ja levittämisen, kunhan samat oikeudet jatkokäytölle taataan uusissakin tuotoksissa.
